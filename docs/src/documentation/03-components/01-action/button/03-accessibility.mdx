# Accessibility

## Button

The Button component has been designed with accessibility in mind.

The component offers flexibility in terms of the HTML element used for the root node. By default, it will render a `button` element.

### As component

The prop `asComponent` can be used to define the HTML element to be rendered.

This prop is optional and if it is not provided, the component will render a default (button) element.

If the `asComponent` prop is set to `"div"`, then the `onClick` prop is not expected (and not recommended to fill in) as it doesn't match the accessibility rules.

### Href

When the `href` prop is defined, the component will render an `a` element by default, no matter what is set to `asComponent`.

## Examples

### Example 1:

Component renders `button` element by default.

```jsx
<Button>Hello World!</Button>
```

renders:

```html
<button>Hello World!</button>
```

### Example 2:

If the `asComponent` prop is set to `"div"`, it is not a good practice to include `onClick` nor `href` attributes.

```jsx
<Button asComponent="div">Hello World!</Button>
```

renders:

```html
<div>Hello World!</div>
```

### Example 3:

A prop `href` is set, so the component renders an `a` element. In this case, rendering this element is prioritized over the others.

```jsx
<Button href="#">Hello World!</Button>
```

renders:

```html
<a href="#">Hello World!</a>
```
